# Tandem D Model Implementation - Follow Up

## Overview
This document tracks the problems encountered and solutions implemented for the tandem D (dual D gene) recombination model in IGoR.

---

## Problem 1: Legacy File Format for Tandem Ds - FIXED ✅

### Issue
The current model_parms.txt file format uses Gene_class enum values to specify the "gene class" for each event. For tandem Ds, we need:
- `D1D2_ins` for the insertion between D1 and D2 genes
- Potentially `VD1_genes` for V-D1 junction, `D2J_genes` for D2-J junction

However, the `Gene_class` enum only supported:
- `V_gene`, `D_gene`, `J_gene` (single gene choices)
- `VD_genes`, `DJ_genes`, `VJ_genes` (junctions between pairs)
- `VDJ_genes`, `Undefined_gene`

When parsing `#Insertion;D1D2_ins;Undefined_side;1;D1D2_ins`, the code called `str2GeneClass("D1D2_ins")` which threw `runtime_error("Unknown Gene_class in str2GeneClass")`.

### Solution
Extended the `Gene_class` enum and related functions to support tandem D-specific gene classes:

1. **Added new enum values in `Utils.h`**:
   - `VD1_genes = 8` - For V-D1 junction events
   - `D1D2_genes = 9` - For D1-D2 junction events  
   - `D2J_genes = 10` - For D2-J junction events

2. **Updated `str2GeneClass()` in `Utils.cpp`** to recognize:
   - `"VD1_genes"` → `VD1_genes`
   - `"D1D2_genes"` → `D1D2_genes`
   - `"D2J_genes"` → `D2J_genes`

3. **Updated `to_string(const Gene_class)`** for reverse conversion

4. **Updated `operator<<` and `operator+` for Gene_class** for logging/debugging

### Files Modified
- `src/igor/Core/Utils.h` - Added new enum values
- `src/igor/Core/Utils.cpp` - Updated conversion functions

## Problem 2: Sequence Type Registration for Tandem Ds - FIXED ✅

### Issue
The `Insertion` and `Dinucl_markov` events look up their sequence type by nickname:
```cpp
int type_id = registry.try_get_type_id(this->nickname);
```

But the `register_tandem_d_types_if_needed()` function was registering types with `_seq` suffix (e.g., `VD1_ins_seq`), while the event nicknames were `VD1_ins` without the suffix. This caused the lookup to fail.

### Solution
Updated `register_tandem_d_types_if_needed()` in `Model_Parms.cpp` to register types with both naming conventions:
```cpp
// Register insertion types (both with and without _seq suffix for compatibility)
auto vd1_ins = registry.register_type("VD1_ins_seq");
registry.register_type("VD1_ins");  // Also register without suffix
// ... same for D1D2_ins, D2J_ins
```

### Files Modified
- `src/igor/Core/Model_Parms.cpp` - Updated registration function

## Problem 4: Mock Binary Event for Second D Gene Probability

### Issue
For a proper tandem D model, we need to model:
1. The probability of having a second D gene (or not)
2. This requires an event that can "skip" the D2 gene choice

Currently, the D2_gene event has a `None` realization, but this is a hack. A cleaner approach would be to introduce a `Binary_event` or `NumberOfD_genes` event that controls whether D2 is processed.

### Proposed Solution
Introduce a new `Rec_Event` subclass for binary/choice events:

```cpp
class Binary_choice : public Rec_Event {
    // Models binary decisions like "has D2?"
    // Realizations: {0, 1} or {false, true}
    // Can be used to skip downstream events
};
```

Or extend `Gene_choice` to support "optional" gene choices where one realization means "skip this segment".

### Alternative (Current Approach)
Keep using the `None` realization in D2_gene event, but properly model its probability distribution based on the number of D genes event.

### Status
⏳ **PENDING** - Current workaround using `None` realization is functional but conceptually unclean. May revisit if needed for inference accuracy.

## Problem 5: Dinucleotide Markov Model Dependencies

### Issue
The dinucleotide Markov model for insertions depends on the nucleotide preceding the insertion:
- VD insertion depends on last V nucleotide
- DJ insertion depends on last D nucleotide
- D1D2 insertion depends on last D1 nucleotide
- D2J insertion depends on last D2 nucleotide (or last D1 if no D2)

**Question**: If D2 is skipped (`None` realization), how does the D2J_dinucl model know which nucleotide to use as the starting state?

### Current Implementation
The `Dinucl_markov::iterate()` method looks up the upstream sequence from `constructed_sequences` and `seq_offsets`:

```cpp
int us_3_off = seq_offsets.at(upstream_seq_type, Three_prime, memory_layer_off_threep);
int prev_nt = int_sequence.at(us_3_off);
```

### Proposed Solution
The topology should ensure that:
1. If D2 exists: D2J_dinucl uses D2's last nucleotide
2. If D2 is None: D2J_dinucl should fall back to D1's last nucleotide

This requires either:
1. **Conditional logic in Dinucl_markov** to check if upstream sequence exists and fall back to alternative
2. **Different model topology** where D2J events are children of a binary "has D2" event
3. **Sequence type registry** that maps D2J to use D1 when D2 is None

### Status
⏳ **UNDER INVESTIGATION** - The current model topology may handle this implicitly through the event graph structure, but needs validation.

## Problem 6: Model Marginals File Format

### Issue
The `model_marginals.txt` file needs to match the events defined in `model_parms.txt`. With the introduction of new event types and nicknames for tandem Ds, the marginals file generation and parsing needs to be verified.

### Current Status
- The model can run without a marginals file (initializes to uniform distribution)
- A template marginals file needs to be generated by running inference or created manually
- The format includes dependency dimensions (`$ParentEvent_size,offset`)

### Workaround
Use the `-set_custom_model` command with only the params file to auto-initialize uniform marginals:
```bash
igor -set_custom_model model_params.txt -generate 100
```

### Status
⏳ **PENDING** - Need to generate a proper template marginals file.
